Security & Privacy

Foundations

Security and privacy on today’s web lean on HTTPS with TLS—confidentiality and integrity are table stakes. HTTP/3 bakes TLS 1.3 into QUIC, reducing handshake time and removing downgrade risks tied to legacy versions. At the same time, features can be abused; for example, poorly controlled parallel streams enabled novel DDoS patterns in HTTP/2. Good hygiene includes enforcing TLS 1.3, enabling HSTS, rate‑limiting abusive patterns, and tracking guidance from browser and IETF security advisories.

Developer Notes

  • Prefer HTTPS everywhere; enforce TLS 1.3 and modern cipher suites.
  • Enable HTTP/3 where supported for security and performance benefits.
  • Use HSTS, OCSP stapling, and robust rate limiting.
